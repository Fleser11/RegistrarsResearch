Audit JSON Files

Audit JSON Files should follow a specific format when created.

  • "info" - Object maping to data about the audit
    • "code" - The code for the audit
    • "code" - The code for the audit
  • "subAudit" - Object mapping sets of types of courses
    • course groups (these names are unspecified but are strings)
      • each of the groups will follow a format of:
      • "cardinality" - (all should be used if all must exist)
      • "courses" - maps to a list of course codes
        • *abstract courses can be declared by prefixing the course code with "abstract_" ex. "abstract_CS4000" for a 4000 level CS course
    • "UReqs" - Object mapping to a path for a university requirements json
